1) Estructurales (Semánticas)

Organizan la página y mejoran accesibilidad/SEO.

Definición: Encabezado de una página o sección.

Sintaxis: <header>...</header>

Ejemplo (código):

<header class="box">
  <h1>Mi Blog</h1>
  <p>Aprende HTML</p>
</header>

Demostración visual:

Mi Blog

Aprende HTML

  • Uso adecuado: pon logo, título, navegación o info del encabezado.
  • Buenas prácticas: evita usarlo solo “por estilo”; úsalo con sentido semántico.

Definición: Contenedor de enlaces de navegación.

Sintaxis: <nav><a href="#">...</a></nav>

<nav class="box">
  <a href="#texto">Texto</a> |
  <a href="#listas">Listas</a>
</nav>
Texto | Listas
  • Uso adecuado: menús principales o secundarios.
  • Buenas prácticas: no metas enlaces irrelevantes; usa listas si hay muchos links.

Definición: Contenido principal del documento (solo uno por página).

Sintaxis: <main>...</main>

Buenas prácticas: no lo repitas, evita colocarlo dentro de <header> o <footer>.

Definición: Agrupa contenido relacionado dentro de una página.

Sintaxis: <section><h2>...</h2>...</section>

Buenas prácticas: casi siempre incluye un título (h2/h3).

Definición: Contenido independiente (post, noticia, entrada).

Sintaxis: <article>...</article>

Buenas prácticas: úsalo para “entradas” reutilizables/compartibles.

Definición: Contenido complementario (sidebar, notas, enlaces relacionados).

Sintaxis: <aside>...</aside>

Buenas prácticas: no lo uses para el contenido principal.

Definición: Pie de página o de sección (autor, copyright, enlaces legales).

Sintaxis: <footer>...</footer>

Buenas prácticas: puede existir en página y también dentro de <article>.

2) Texto y Contenido

Etiquetas para títulos, párrafos y énfasis.

Definición: Jerarquía de títulos (h1 el más importante).

Sintaxis: <h2>Subtítulo</h2>

<h2>Título de sección</h2>
<h3>Subtítulo</h3>

Título de sección

Subtítulo
  • Buenas prácticas: no saltes niveles sin razón (h2 → h4).
  • Uso adecuado: estructura el contenido como un esquema.

Definición: Párrafo.

Sintaxis: <p>Texto...</p>

Buenas prácticas: no uses <br> para “hacer párrafos”.

Definición: Contenedor en línea para aplicar estilos o agrupar texto.

Sintaxis: <span class="badge">Nuevo</span>

Etiqueta Nuevo

Buenas prácticas: úsalo cuando NO exista una etiqueta semántica mejor.

Definición: Énfasis semántico (importancia y énfasis).

Sintaxis: <strong>Importante</strong> y <em>énfasis</em>

Esto es importante y esto tiene énfasis.

Buenas prácticas: no lo uses solo para “negrita/cursiva” sin sentido.

Definición: <br> salto de línea; <hr> separación temática.

Línea 1
Línea 2
Sección separada

Buenas prácticas: evita abusar de <br> para maquetar.

3) Listas

Para agrupar elementos relacionados.

Definición: Lista desordenada, ordenada y elemento de lista.

<ul>
  <li>HTML</li>
  <li>CSS</li>
</ul>

<ol>
  <li>Planear</li>
  <li>Construir</li>
</ol>
  • HTML
  • CSS
  1. Planear
  2. Construir
  • Buenas prácticas: usa <li> siempre dentro de ul/ol.

Definición: Lista de definición: término (dt) y descripción (dd).

HTML
Lenguaje de marcado para estructurar páginas web.

Buenas prácticas: ideal para glosarios y pares término/definición.

4) Formularios (Obligatorio)

Incluye etiquetas: form, label, input, textarea, select, option, button, fieldset, legend.

Definición: Agrupa controles para enviar datos.

Buenas prácticas: usa label asociado con for y id; valida en cliente y servidor.

Ejemplo (código):

<form id="contactForm">
  <fieldset>
    <legend>Contacto</legend>

    <label for="name">Nombre</label>
    <input id="name" name="name" type="text" required>

    <label for="email">Email</label>
    <input id="email" name="email" type="email" required>

    <label for="topic">Tema</label>
    <select id="topic" name="topic">
      <option value="html">HTML</option>
      <option value="css">CSS</option>
    </select>

    <label for="msg">Mensaje</label>
    <textarea id="msg" name="msg" rows="3" required></textarea>

    <button type="submit">Enviar</button>
  </fieldset>
</form>

5) Tablas

Para datos tabulares (no para maquetar).

Definición: Estructura datos en filas/columnas.

<table>
  <thead>
    <tr><th>Etiqueta</th><th>Uso</th></tr>
  </thead>
  <tbody>
    <tr><td>p</td><td>Párrafo</td></tr>
  </tbody>
</table>
EtiquetaUso
<p>Párrafo
<img>Imagen
  • Buenas prácticas: usa <th> para encabezados; no uses tablas para layout.

6) Multimedia

Imágenes, audio, video e incrustaciones.

Definición: Inserta una imagen.

Sintaxis: <img src="..." alt="Descripción">

Buenas prácticas: siempre usa alt (accesibilidad/SEO) y tamaños adecuados.

Definición: Reproductor de audio con fuentes alternativas.

<audio controls>
  <source src="audio.mp3" type="audio/mpeg">
</audio>

Demo: usa un archivo real en tu repo (por ejemplo assets/audio.mp3). 

<video controls width="480">
  <source src="video.mp4" type="video/mp4">
</video>

Buen tip: incluye controls y compresión adecuada del video.

Definición: Incrusta contenido externo (mapas, videos, etc.).

Buenas prácticas: revisa permisos (allow), rendimiento y seguridad.
